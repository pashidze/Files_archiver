## Files_archiver

## Описание
Программа, предназначенная для рекурсивного обхода указанной директории и архивирования всех найденных файлов.

## Реализованные возможности

- Передача программе пути к директории с помощью параметра при запуске или в консоли;
- Рекурсивный обход указанной директории и добавление всех найденных файлов в очередь;
- Многопоточное сжатие найденных файлов (используется библиотека с открытым исходным кодом libzip);
- Перенос сжатых файлов в директорию с исполняемым файлом программы.

## Технологический стек

- **С++20;**
- **Visual Studio 2022;**
- **CMake 4.1.1;**
- **vcpkg;**
- **libzip**

## Установка и запуск

### Windows

1. Клонировать репозиторий вместе с подмодулями
```shell
git clone --recurse-submodules https://github.com/pashidze/Files_archiver.git
```

2. Установить CMake, если он отсутствует на устройстве
```shell
https://cmake.org/
```

3. Перейти в директорию с проектом
```shell
cd Files_archiver
```

4. Выполнить сборку проекта командами. Важно: сборка может занимать достаточно продолжительное время, так как подгружаются файлы зависимостей
```shell
cmake --preset windows
cmake --build out/build/windows --config Release
```

5. Перейти к исполняемому файлу проекта
```shell
cd out\build\windows\Files_archiver\Release
```

6. Выполнить запуск приложения (с параметром или без)
```shell
.\Files_archiver.exe 'C:Path\To\Directory'
.\Files_archiver.exe
```

6.1. В случае запуска приложения без параметра, указать путь к директории после сообщения 
```shell
Укажите директорию: C:Path\To\Directory
```

7. Важно: Сжатие файлов большого объёма может занимать продолжительное время

### Linux

1. Клонировать репозиторий вместе с подмодулями
```shell
git clone --recurse-submodules https://github.com/pashidze/Files_archiver.git  
```

2. Установить CMake, компилятор и систему сборки Ninja, если они отсутствует на устройстве
```shell
sudo apt update
sudo apt install cmake
sudo apt install build-essential
sudo apt install ninja-build
```

3. Перейти в директорию с проектом
```shell
cd Files_archiver
```

4. Выполнить сборку проекта командами. Важно: сборка может занимать достаточно продолжительное время, так как подгружаются файлы зависимостей
```shell
cmake --preset linux
cmake --build out/build/linux --config Release
```

5. Перейти к исполняемому файлу проекта
```shell
cd out/build/linux/Files_archiver
```

6. Выполнить запуск приложения (с параметром или без)
```shell
./Files_archiver 'Path/To/Directory'
./Files_archiver
```

6.1. В случае запуска приложения без параметра, указать путь к директории после сообщения 
```shell
Укажите директорию: Path/To/Directory
```

7. Важно: Сжатие файлов большого объёма может занимать продолжительное время
